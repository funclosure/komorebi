# Seal Rail Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the panel's bottom text actions (another tree / copy the prompt / copy this light) with a vertical rail of three rounded-rect "seal" buttons floating right of the sticky panel, docking into the panel as a horizontal row on narrow screens.

**Architecture:** Pure page-chrome change in `index.html`: new `.rail`/`.seal` CSS + one `@media (max-width:1219px)` docking rule; the two old panel groups are replaced by the rail markup plus a hidden correctness block (sr-only take-note keeping `aria-describedby` valid, and the clipboard-fallback textarea). The buttons keep ids `regrow`, `copyPrompt`, `copyLink`, so every existing script binding survives; the only script edit is one line in the playground `flash()` (`'copied ✓'` → `'✓'`) because the long label cannot fit a seal face.

**Tech Stack:** Plain CSS/HTML/JS in the self-contained `index.html`. Verification via local `python3 -m http.server` + Playwright MCP tools. Deploy via `git push` (Pages rebuilds from main).

## Global Constraints

- The `<script id="engine">` block must not be modified in any way.
- The ONLY permitted `<script id="playground">` edit: the single `flash()` line specified in Task 1 Step 4.
- Element ids `regrow`, `copyPrompt`, `copyLink`, `takeNote`, `copyFallback` must exist after the change (scripts and ARIA bind to them).
- Everything stays inline in `index.html`; no external requests (SVG icons are inline markup).
- Spec: `docs/superpowers/specs/2026-07-13-seal-rail-design.md`.
- Commit messages end with `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.
